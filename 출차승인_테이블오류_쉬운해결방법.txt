출차 승인 테이블 오류 쉬운 해결 방법
=====================================

오류 문구
---------
Could not find the table 'public.departure_checks' in the schema cache

또는

function public.has_permission(unknown) does not exist

오류 원인
---------
Supabase에 출차 점검·승인 내용을 저장하는 departure_checks 테이블이
아직 만들어지지 않은 상태입니다.

기존 배송원 계정, 거래처, 발주, 거래명세, 전체 배송 완료 자료가
삭제된 것은 아닙니다.


해결 순서
---------
1. Supabase 사이트에 로그인합니다.

2. 장백수산 프로젝트를 선택합니다.

3. 왼쪽 메뉴에서 SQL Editor를 누릅니다.

4. New query를 누릅니다.

5. 아래 파일을 열어 전체 내용을 복사합니다.
   Supabase_출차승인테이블_오류수정_SQL.txt

   이전에 복사했던 SQL이 아니라 수정된 최신 파일 전체를 다시 복사해야 합니다.

6. SQL Editor의 빈 화면에 붙여 넣습니다.

7. 오른쪽 아래 Run 버튼을 한 번 누릅니다.

8. Results에 아래 문구가 나오면 성공입니다.
   public.departure_checks

9. 관리자 페이지로 돌아갑니다.

10. 화면 위쪽 ‘업데이트 새로고침’을 누릅니다.

11. 다시 ‘출차 승인’을 누릅니다.


중요
----
- GitHub 코드를 다시 수정할 필요는 없습니다.
- Vercel을 다시 배포할 필요도 없습니다.
- SQL은 한 번만 실행하면 됩니다.
- 같은 SQL을 실수로 다시 실행해도 기존 자료를 삭제하지 않도록 작성했습니다.
- 최신 SQL은 has_permission 함수가 없는 이전 Supabase 구조에서도 실행됩니다.
