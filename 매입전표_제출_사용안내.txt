장백수산 중앙관리 V2.3 · 호차별 매입전표 제출 사용 안내

[기능 목적]
각 호차 배송원이 카카오톡 발주 내용을 배송원 웹앱에 모두 입력한 뒤,
해당 호차의 제품별 매입 수량을 회사 구매 담당자에게 중앙으로 전달하는 기능입니다.

[배송원 사용 순서]
1. 카카오톡으로 받은 거래처 발주를 배송원 웹앱에 모두 입력합니다.
2. 배송원 메인 화면의 ‘당일 매입발주서’를 누릅니다.
3. 거래처별 품목과 제품별 총 매입 수량을 확인합니다.
4. 구매 담당자에게 전달할 내용이 있으면 ‘구매 담당자 전달 메모’에 입력합니다.
5. ‘구매 담당자에게 매입전표 제출’을 누릅니다.
6. 확인창에서 확인을 누르면 관리자 페이지로 중앙 전달됩니다.
7. 화면에 ‘구매 담당자 제출 완료’가 표시되면 완료입니다.

[발주를 수정한 경우]
1. 매입전표 제출 후 발주 품목이나 수량을 수정할 수 있습니다.
2. 수정 즉시 기존 제출 상태가 ‘발주 내용 변경 · 매입전표 다시 제출 필요’로 바뀝니다.
3. 당일 매입발주서를 다시 엽니다.
4. 변경된 내용을 확인하고 ‘변경된 매입전표 다시 제출’을 누릅니다.
5. 제출 차수는 2차, 3차 순서로 자동 기록됩니다.

[구매 담당자·관리자 사용 순서]
1. 관리자 페이지에 로그인합니다.
2. 위쪽 메뉴에서 ‘매입전표’를 누릅니다.
3. 날짜를 선택합니다.
4. 호차별 제출 완료, 재제출 필요, 제출 대기 상태를 확인합니다.
5. ‘호차 전표 상세’를 누르면 해당 호차의 거래처별 수량을 확인할 수 있습니다.
6. 아래 ‘구매 담당자 통합 매입 수량’에서 모든 제출 완료 호차의 제품 합계를 확인합니다.
7. ‘매입전표 Excel’을 누르면 통합 매입전표, 호차별 매입전표, 제출 현황이 저장됩니다.

[제출 상태 의미]
- 제출 완료: 현재 발주 내용과 제출 전표가 일치합니다.
- 변경·재제출 필요: 제출 후 발주 내용이 수정되었습니다.
- 작성·제출 대기: 발주는 입력했지만 구매 담당자에게 제출하지 않았습니다.
- 발주 없음: 해당 날짜에 입력된 발주 품목이 없습니다.

[중요한 집계 원칙]
- 취소된 배송은 매입전표에서 제외됩니다.
- 같은 제품이라도 규격, 원산지, 작업, 단위, 밴드 조건이 다르면 별도로 표시됩니다.
- 관리자 통합 매입 수량에는 ‘제출 완료’ 호차만 반영됩니다.
- ‘변경·재제출 필요’ 호차는 다시 제출하기 전까지 통합 합계에서 제외됩니다.

[업데이트 방법]
- GitHub의 driver.html과 admin.html 두 파일을 모두 새 버전으로 교체합니다.
- Supabase SQL과 환경변수는 다시 설정할 필요가 없습니다.
- GitHub 저장 후 Vercel 최신 배포가 Ready가 되면 두 페이지를 새로고침합니다.
