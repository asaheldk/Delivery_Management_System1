장백수산 관리자 V2.4.3
배송원 전체 정보 수정·삭제 사용 안내
===================================

이번에 추가된 기능
----------------
1. 최고 관리자가 배송원 로그인 아이디 직접 변경
2. 배송원 배정 호차·이름·연락처·계정 상태 수정
3. 아이디·호차 변경 또는 퇴사 사유 기록
4. 배송원 비밀번호 초기화
5. 최고 관리자만 배송원 계정 완전 삭제
6. 아이디 변경·배송원 삭제 작업을 감사 기록에 저장


1. 프로그램 적용
--------------
1) V2.4.3 전체 ZIP의 압축을 풉니다.
2) 압축을 푼 폴더 안의 모든 파일과 폴더를 GitHub 저장소에 덮어씁니다.
3) 특히 다음 파일이 반드시 올라가야 합니다.

- admin.html
- api/admin/update-driver.js
- api/admin/delete-driver.js
- api/admin/auth.js
- package.json
- package-lock.json

4) Vercel에서 최신 배포가 초록색 Ready가 될 때까지 기다립니다.
5) 관리자 페이지에 로그인합니다.
6) 오른쪽 위 '업데이트 새로고침' 버튼을 누릅니다.
7) 왼쪽 위에 OPERATIONS CENTER V2.4.3이 보이면 적용 완료입니다.

이번 기능은 Supabase 표를 추가하지 않으므로 별도 SQL 실행은 필요 없습니다.


2. 배송원 전체 정보 수정
----------------------
1) 상단 메뉴에서 '배송원·출차'를 누릅니다.
2) 수정할 배송원의 오른쪽 '전체 정보 수정'을 누릅니다.
3) 다음 내용을 직접 수정합니다.

- 배송원 로그인 아이디
- 배정 호차
- 배송원 이름
- 연락처
- 계정 상태
- 아이디·호차 변경 또는 퇴사 사유

4) '전체 정보 저장'을 누릅니다.


3. 배송원 아이디 변경 시 중요사항
--------------------------------
예시:

기존 아이디: driver01
새 아이디: driver02

저장하면:
- Supabase 로그인 계정도 driver02로 함께 변경됩니다.
- 배송원 프로필의 아이디도 driver02로 변경됩니다.
- 기존 driver01 아이디로는 로그인할 수 없습니다.
- 비밀번호는 기존 비밀번호가 그대로 유지됩니다.

최고 관리자는 새 아이디를 해당 배송원에게 바로 전달해야 합니다.

다른 계정에서 이미 사용하는 아이디는 저장할 수 없습니다.
아이디는 영문 소문자·숫자·밑줄·빼기만 사용할 수 있습니다.


4. 호차 변경
-----------
1) '배정 호차'에 새 호차를 입력합니다.
2) 변경 사유를 입력합니다.
3) 저장합니다.

저장하면:
- 기존 호차 배정 이력이 종료됩니다.
- 새 호차 배정 이력이 생성됩니다.
- 이미 다른 사용 중 배송원에게 배정된 호차는 사용할 수 없습니다.


5. 배송원 퇴사·일시 정지
----------------------
배송 기록을 보존해야 한다면 삭제하지 말고 다음 방법을 권장합니다.

1) '전체 정보 수정'을 누릅니다.
2) 계정 상태를 '퇴사·사용 정지'로 선택합니다.
3) 퇴사 또는 정지 사유를 입력합니다.
4) 저장합니다.

사용 정지 계정은 로그인할 수 없지만 기존 기록은 보존됩니다.


6. 배송원 비밀번호 초기화
-----------------------
1) '전체 정보 수정'을 누릅니다.
2) 아래쪽 '비밀번호 초기화'를 누릅니다.
3) 새로운 임시 비밀번호를 8자리 이상 입력합니다.
4) 변경된 비밀번호를 해당 배송원에게 개인적으로 전달합니다.


7. 배송원 완전 삭제
------------------
배송원 삭제는 최고 관리자만 할 수 있습니다.

1) '배송원·출차'에서 삭제할 배송원을 찾습니다.
2) 오른쪽 빨간색 '삭제'를 누릅니다.
3) 삭제 경고창에서 배송원 이름을 확인합니다.
4) 계속 진행하면 배송원 아이디를 다시 입력하라는 창이 나옵니다.
5) 화면에 표시된 아이디를 정확히 입력합니다.

삭제되는 내용:
- 배송원 로그인 계정
- 배송원 프로필
- 해당 배송원의 중앙 백업·당일 상태 자료
- 해당 배송원의 호차 배정 이력

해당 배송원에게 배정됐던 거래처는 미배정 상태로 변경됩니다.

주의:
- 삭제는 되돌릴 수 없습니다.
- 배송 기록 또는 인수인계 기록이 필요하면 '사용 정지'를 사용하세요.
- 최고 관리자가 아닌 계정은 삭제 버튼이 보이지 않으며,
  주소를 직접 호출해도 서버에서 삭제가 거부됩니다.


8. 화면에서 확인할 버튼
---------------------
최고 관리자로 로그인한 경우 배송원마다 다음 버튼이 보입니다.

- 배정 이력
- 전체 정보 수정
- 삭제

배차 관리자는 다음 기능을 사용할 수 있습니다.

- 배정 이력
- 호차·이름·연락처·상태 수정

배차 관리자는 배송원 로그인 아이디를 변경하거나
배송원 계정을 삭제할 수 없습니다.


9. 버튼이 보이지 않을 때
----------------------
1) Vercel 최신 배포가 Ready인지 확인합니다.
2) 관리자 화면 왼쪽 위 버전이 V2.4.3인지 확인합니다.
3) 오른쪽 위 '업데이트 새로고침'을 누릅니다.
4) 로그아웃 후 최고 관리자 계정으로 다시 로그인합니다.
