장백수산 전체 배송 완료 기능 V2.4.4 사용 안내
================================================

이번 업데이트의 핵심
--------------------
1. 각 점포마다 누르던 ‘하차 완료’ 버튼을 삭제했습니다.
2. 점포별 하차 완료 시간은 관리자 화면에 표시하지 않습니다.
3. 배송원이 하루 마지막 배송까지 마친 뒤 한 번만 ‘오늘 전체 배송 완료’를 누릅니다.
4. 별도 전화나 카카오톡 업무 보고 없이 관리자 화면에 전체 완료 상황이 전달됩니다.


1. 배송원 웹앱 사용 방법
-----------------------
① 평소처럼 배송 순서에 따라 이동합니다.

② 각 거래처의 ‘물품·명세 확인’을 누르면 다음 내용을 확인할 수 있습니다.
   - 어종과 수량
   - 수조 또는 박스 운반 위치
   - 거래명세서
   - 전미수
   - 점포 특이사항

   점포마다 ‘배송 완료’를 누를 필요는 없습니다.

③ 오늘 배정된 마지막 배송까지 모두 끝냅니다.

④ 배송원 웹앱 오늘 화면 위쪽의
   ‘오늘 배송을 모두 마쳤습니까?’ 영역에서
   ‘전체 배송 완료’를 누릅니다.

⑤ 상황에 따라 아래 두 버튼 중 하나를 누릅니다.

   [문제 없이 전체 배송 완료]
   - 추가 보고할 문제가 없을 때 누릅니다.
   - 전체 완료 시간과 ‘문제 없음’ 상태가 관리자에게 전달됩니다.

   [배송 완료 · 관리자 확인 필요]
   - 배송은 끝났지만 회사가 확인해야 할 내용이 있을 때 누릅니다.
   - 예: 긴급 추가 발주 미처리, 오배송 확인 필요, 점포 연락 필요
   - 버튼을 누른 뒤 관리자에게 전달할 내용을 입력합니다.

⑥ 완료 후에는 당일 자료가 실수로 바뀌지 않도록 잠깁니다.

⑦ 잘못 완료했거나 추가 발주가 생기면
   ‘완료 내용 보기·취소’ → ‘전체 완료 취소 · 다시 수정’을 누릅니다.
   취소 사유를 입력하면 당일 자료를 다시 수정할 수 있습니다.


2. 관리자 웹앱 확인 방법
-----------------------
① 관리자 페이지에 로그인합니다.

② 상단 메뉴에서 ‘오늘 관제’를 누릅니다.

③ ‘호차별 전체 배송 완료 현황’을 확인합니다.

   각 호차마다 다음 내용이 표시됩니다.
   - 전체 완료 또는 진행 중
   - 전체 완료 시간
   - 문제 없음 또는 관리자 확인 필요
   - 당일 배정 점포 수
   - 긴급 미처리 건수
   - 미해결 오류 건수
   - 배송원이 작성한 완료 메모
   - 마지막 중앙 동기화 시간

④ ‘전체 상황 상세’를 누르면 해당 호차의 자세한 내용을 볼 수 있습니다.

중요:
- 점포별 하차 완료 시간은 수집·표시하지 않습니다.
- 거래처 목록에는 배정 순서와 품목·박스 수만 표시됩니다.
- 배송원을 압박하는 점포별 시간 비교 기능은 없습니다.


3. GitHub·Vercel 적용 방법
-------------------------
① GitHub 저장소에서 기존 driver.html을 엽니다.

② 아래 파일의 전체 내용을 복사해 driver.html에 붙여 넣습니다.
   배송원_페이지_V9_4_전체배송완료_복사붙여넣기.txt

③ 기존 admin.html을 엽니다.

④ 아래 파일의 전체 내용을 복사해 admin.html에 붙여 넣습니다.
   관리자_페이지_V2_4_4_전체배송완료현황_복사붙여넣기.txt

⑤ package.json도 ZIP 안의 최신 파일로 교체합니다.

⑥ GitHub에서 Commit changes를 눌러 저장합니다.

⑦ Vercel이 자동 배포를 마칠 때까지 기다립니다.
   상태가 Ready가 되면 배포 완료입니다.

⑧ 관리자 페이지와 배송원 페이지에서
   ‘업데이트 새로고침’ 버튼을 한 번 누릅니다.

⑨ 배송원 한 명으로 먼저 시험합니다.
   - 점포별 완료 버튼이 없는지
   - 전체 배송 완료 버튼이 보이는지
   - 관리자 현황에 완료 시간이 나오는지
   - 완료 취소가 되는지
   확인한 뒤 전체 배송원에게 적용합니다.


4. 데이터베이스 작업 여부
-------------------------
이번 업데이트는 기존 driver_state_snapshots와 driver_daily_status를 사용합니다.
따라서 Supabase SQL Editor에서 새로운 SQL을 실행할 필요가 없습니다.


5. 권장 시험 순서
-----------------
1) 테스트 배송 1건 등록
2) 거래처 화면에서 물품·명세 확인
3) 점포별 완료 버튼이 없는지 확인
4) ‘문제 없이 전체 배송 완료’ 실행
5) 관리자 오늘 관제에서 전체 완료 시간 확인
6) 배송원 화면에서 전체 완료 취소
7) ‘관리자 확인 필요’로 다시 완료하고 메모 확인
