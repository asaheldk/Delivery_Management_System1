장백수산 관리자 페이지 V2.2 · 당일 제품별 출고량 사용 안내

[1. 어디에서 확인하나요?]
1) 관리자 페이지에 로그인합니다.
2) 화면 위 메뉴에서 “출고량 분석”을 누릅니다.
3) 오른쪽 위 날짜에서 확인할 날짜를 선택합니다.

[2. 화면에서 바로 보이는 내용]
- 출고 제품: 선택한 날짜에 등록된 전체 제품 종류 수
- 총 마릿수: 활어의 개별 계근 입력 개수 합계
- 총 kg: 개별 계근 합계 → 직접 입력한 실제 중량 → kg 주문수량 순으로 집계
- 출고 호차: 해당 날짜 자료가 있는 차량 수
- 계근 미확정: 활어 계근 완료 버튼을 아직 누르지 않은 품목 건수
- 최근 동기화: 배송원 휴대폰 자료가 중앙 서버에 마지막 저장된 시간

[3. 제품별 전체 합계]
- 광어, 도미, 농어 등 같은 제품을 1호차부터 전체 호차까지 합산합니다.
- 같은 “광어”라도 활어와 필렛 제품은 서로 다른 제품으로 구분합니다.
- 활어는 “총 몇 마리 / 총 몇 kg”으로 표시합니다.
- 우니 등 냉동제품은 “몇 팩”, 패류는 망·봉지·박스 등 입력 단위로 표시합니다.
- 취소한 배송은 합계에서 자동 제외됩니다.

[4. 호차별·제품별 출고량]
화면 아래 “호차별·제품별 출고량” 표에서 다음 내용을 확인합니다.
- 호차와 배송원 이름
- 제품명과 구분
- 제품별 총 마릿수
- 제품별 총 kg
- 팩·박스·망·봉지 등 단위별 수량
- 납품처 수
- 계근 확정 여부

[5. 원하는 자료만 보는 방법]
- 검색: 제품명, 거래처, 배송원 이름, 호차를 입력합니다.
- 전체 호차: 특정 호차만 선택할 수 있습니다.
- 출고 대상 전체: 준비 중과 배송 완료 자료를 모두 표시합니다.
- 납품 완료만: 배송 완료 처리한 자료만 표시합니다.
- 준비·배송 중: 아직 완료되지 않은 자료만 표시합니다.

[6. 엑셀 저장]
오른쪽 위 “출고량 Excel”을 누르면 다음 3개 시트가 한 파일로 저장됩니다.
1) 제품별 전체합계
2) 호차별 제품합계
3) 출고 상세

[7. 숫자가 보이지 않을 때]
1) 배송원이 해당 날짜 발주·계근 내용을 저장했는지 확인합니다.
2) 배송원 휴대폰이 인터넷에 연결되어 중앙 서버 동기화가 되었는지 확인합니다.
3) 관리자 화면의 날짜가 배송일과 같은지 확인합니다.
4) 화면의 “최근 동기화” 시간이 오래되었다면 배송원 화면을 다시 열어 저장합니다.

[8. 업데이트 방법]
- 기존 GitHub 저장소의 admin.html만 새 파일로 교체하면 됩니다.
- driver.html과 Supabase SQL은 이번 기능 때문에 다시 교체하거나 실행할 필요가 없습니다.
- GitHub 반영 후 Vercel이 자동 배포되면 관리자 페이지를 새로고침합니다.
