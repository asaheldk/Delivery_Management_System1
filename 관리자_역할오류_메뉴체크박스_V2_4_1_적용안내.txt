장백수산 관리자 V2.4.1 적용 안내
=================================

이번 수정 내용
--------------
1. 관리자 생성 때 나오던 아래 빨간 오류를 수정합니다.
   new row for relation "profiles" violates check constraint "profiles_role_check"

2. 새 관리자 또는 기존 관리자의 역할을 선택하면
   바로 아래에 추천 메뉴 체크박스가 자동으로 표시됩니다.

3. 최고 관리자는 추천값을 그대로 쓰거나,
   필요한 메뉴만 직접 체크·해제해 저장할 수 있습니다.

4. 이미 등록된 이메일을 다시 입력했을 때
   이해하기 쉬운 한국어 안내가 표시됩니다.


1단계 · 관리자 생성은 잠시 멈추기
---------------------------------
현재 빨간 오류가 보이는 상태에서는 같은 내용을 반복해서 저장하지 마세요.
먼저 아래 2단계의 Supabase SQL을 실행해야 합니다.


2단계 · Supabase 역할 오류 수정
------------------------------
1. Supabase에 로그인합니다.
2. 장백수산 프로젝트를 엽니다.
3. 왼쪽 메뉴에서 SQL Editor를 누릅니다.
4. New query를 누릅니다.
5. 다음 파일을 엽니다.

   Supabase_V2_4_1_관리자역할_긴급수정_SQL.txt

6. 파일 안의 내용을 처음부터 끝까지 모두 복사합니다.
7. Supabase SQL 입력 칸에 붙여넣습니다.
8. 오른쪽 아래 또는 위쪽의 Run 버튼을 누릅니다.
9. Results에 profiles_role_v2_check가 보이면 성공입니다.

주의:
- SQL에서 SUPER_ADMIN처럼 대문자로 바꾸지 마세요.
- 파일 내용을 그대로 실행하세요.
- 같은 SQL을 여러 번 실행해도 됩니다.

- admin.html
- api/admin/auth.js
- api/admin/create-staff.js
- api/admin/update-role.js
- package.json
- package-lock.json

GitHub에서 파일 하나씩 바꾸는 경우:
1. 바꿀 파일을 누릅니다.
2. 연필 모양 Edit 버튼을 누릅니다.
3. 기존 내용을 모두 지웁니다.
4. V2.4.1의 같은 이름 파일 내용을 전부 붙여넣습니다.
5. Commit changes를 누릅니다.

api 폴더 안의 파일은 반드시 api/admin 폴더 안에 그대로 있어야 합니다.


4단계 · Vercel 새 배포 확인
--------------------------

6단계 · 메뉴 체크박스 확인
-------------------------
1. 최고 관리자 계정으로 로그인합니다.
2. 위쪽 메뉴에서 권한·감사를 누릅니다.
3. + 관리자 버튼을 누릅니다.
4. 역할을 눌러 다음 중 하나를 선택합니다.

   최고 관리자
   배차 관리자
   회계 관리자
   현장 관리자
   조회 전용

5. 역할 아래에 '사용할 메뉴 선택' 체크박스가 바로 보입니다.
6. 역할을 바꿀 때마다 추천 메뉴 체크가 즉시 바뀌는지 확인합니다.
7. 필요한 메뉴는 추가로 체크하고, 필요 없는 메뉴는 체크를 풉니다.
8. 실수했으면 '추천으로 되돌리기'를 누릅니다.


7단계 · 새 관리자 계정 만들기
----------------------------
각 입력 칸은 아래처럼 작성합니다.

- 로그인 이메일:
  기존 최고 관리자와 다른 이메일을 입력합니다.
  실제 운영할 때는 관리자 한 명당 서로 다른 이메일을 권장합니다.

- 관리자 아이디:
  영문 소문자, 숫자, 밑줄(_), 빼기(-)만 사용합니다.
  예: dispatch01

- 담당자 이름:
  실제 사용자 이름을 입력합니다.
  예: 홍길동

- 임시 비밀번호:
  8자리 이상 입력합니다.
  다른 사람에게 노출되지 않게 개인적으로 전달합니다.

- 역할:
  업무에 맞는 역할을 선택합니다.

- 사용할 메뉴:
  자동으로 체크된 추천 메뉴를 확인한 뒤 필요하면 바꿉니다.

마지막으로 '권한 저장'을 누릅니다.


중요 · 현재 화면에 입력했던 이메일
---------------------------------
첨부 화면의 asaheldk@gmail.com 주소가 현재 최고 관리자 로그인 이메일이라면,
새 관리자 계정에는 같은 이메일을 사용할 수 없습니다.

새 관리자의 실제 이메일을 입력해야 합니다.
같은 이메일을 다시 쓰면 '이미 사용 중인 이메일' 오류가 나옵니다.

정상 작동 최종 확인
------------------
1. 새 관리자가 본인 이메일과 임시 비밀번호로 로그인합니다.
