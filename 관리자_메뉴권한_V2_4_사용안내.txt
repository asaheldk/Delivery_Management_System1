장백수산 관리자 V2.4
역할별 메뉴 체크박스 사용 안내

==================================================
1. 이번 업데이트에서 달라진 점
==================================================

새 관리자 계정을 만들거나 기존 관리자 권한을 변경할 때
“사용할 메뉴 선택” 체크박스가 표시됩니다.

역할을 선택하면 해당 역할에 맞는 추천 메뉴가 자동으로 체크됩니다.
최고 관리자는 추천 내용을 그대로 사용하거나 체크를 직접 바꿀 수 있습니다.

저장 후 해당 관리자가 다시 로그인하면
체크한 메뉴만 화면 위쪽에 표시됩니다.


==================================================
2. 업데이트 순서 — 반드시 이 순서대로 진행
==================================================

1) Supabase에 접속합니다.
2) 왼쪽 메뉴에서 SQL Editor를 누릅니다.
3) New query를 누릅니다.
4) “Supabase_V2_4_메뉴권한_업데이트_SQL.txt” 내용을 전부 복사합니다.
5) SQL Editor에 붙여넣습니다.
6) Run을 누릅니다.
7) 아래 Results에 관리자 계정 목록이 나오면 정상입니다.
8) GitHub 저장소의 프로그램 파일 전체를 V2.4 파일로 교체합니다.
9) Vercel에서 자동 배포가 끝날 때까지 기다립니다.
10) 관리자 페이지를 새로고침합니다.

주의:
SQL을 먼저 실행하지 않고 프로그램 파일만 올리면
관리자 계정 생성 시 menu_permissions 열이 없다는 오류가 날 수 있습니다.


==================================================
3. 새 관리자 생성 방법
==================================================

1) 관리자 페이지에 최고 관리자 계정으로 로그인합니다.
2) 위쪽에서 “권한·감사” 메뉴를 누릅니다.
3) “＋ 관리자” 버튼을 누릅니다.
4) 이메일, 관리자 아이디, 담당자 이름, 임시 비밀번호를 입력합니다.
5) 역할을 선택합니다.
6) 역할에 맞는 추천 메뉴가 자동으로 체크되는지 확인합니다.
7) 필요 없는 메뉴는 체크를 해제합니다.
8) 추가로 필요한 메뉴는 체크합니다.
9) “권한 저장”을 누릅니다.
10) 새 관리자가 로그인하면 체크한 메뉴만 표시됩니다.


==================================================
4. 기존 관리자 메뉴 변경 방법
==================================================

1) “권한·감사” 메뉴를 누릅니다.
2) 관리자 계정 목록에서 변경할 사람의 “권한 변경”을 누릅니다.
3) 역할과 체크된 메뉴를 확인합니다.
4) 필요한 메뉴를 체크하거나 해제합니다.
5) “권한 저장”을 누릅니다.
6) 해당 관리자는 로그아웃 후 다시 로그인하거나 페이지를 새로고침합니다.

==================================================
6. 화면 버튼 설명
==================================================

[추천으로 되돌리기]
현재 선택한 역할의 추천 메뉴로 체크 상태를 다시 맞춥니다.

[사용 가능 메뉴 전체 선택]
현재 역할에서 사용할 수 있는 메뉴를 모두 체크합니다.

[전체 해제]
모든 메뉴 체크를 해제합니다.
단, 저장할 때는 메뉴를 최소 1개 이상 선택해야 합니다.

[추천 표시]
선택한 역할에 기본적으로 권장되는 메뉴입니다.
추천 표시는 참고용이며 최고 관리자가 체크를 바꿀 수 있습니다.


==================================================
7. 중요 권한 원칙
==================================================

1) “권한·감사” 메뉴는 최고 관리자만 실제 사용할 수 있습니다.
2) 메뉴 체크는 화면에 보일 메뉴를 결정합니다.
3) 등록·수정·삭제 같은 실제 작업 권한은 선택한 역할 기준으로 제한됩니다.
4) 조회 전용 계정은 메뉴를 보더라도 내용을 변경할 수 없습니다.
5) 본인 최고 관리자 계정은 화면에서 실수로 권한을 낮출 수 없습니다.
6) 역할과 메뉴 권한을 변경한 기록은 감사 기록에 남습니다.


==================================================
8. 문제가 생겼을 때
==================================================

[저장할 때 menu_permissions 오류]
- Supabase_V2_4_메뉴권한_업데이트_SQL.txt를 먼저 실행했는지 확인합니다.

[저장했는데 메뉴가 바로 바뀌지 않음]
- 해당 관리자 계정에서 로그아웃합니다.
- 다시 로그인합니다.
- 그래도 같으면 브라우저를 완전히 종료한 뒤 다시 접속합니다.

[권한·감사 메뉴가 안 보임]
- 최고 관리자 역할인지 확인합니다.
- 해당 계정의 메뉴 권한에 “권한·감사”가 체크됐는지 확인합니다.

[메뉴를 하나도 선택하지 못함]
- 최소 1개 메뉴는 반드시 체크해야 저장됩니다.
