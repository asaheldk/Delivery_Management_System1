장백수산 배송 통합 관리자 V2.5.2
날짜 분리·회계 자동 연동·직접 수정 사용 안내

==================================================
1. 이번에 수정된 문제
==================================================

1) 7월 23일 배송 내용이 다른 날짜에도 반복 표시되던 문제

- 관리자 화면이 선택한 날짜의 배송만 확인하도록 수정했습니다.
- 선택한 날짜에 실제 배송, 전체 배송 완료, 출차 승인, 긴급 발주가 있는 배송원만
  ‘오늘 통합 관제’의 당일 업무 화면에 표시됩니다.
- 예: 2026-07-25를 선택했는데 2026-07-23 배송만 있는 경우,
  7월 23일 배송 내용은 7월 25일 화면에 나오지 않습니다.

2) 배송 이력이 있는데 회계·보고가 비어 있던 문제

- 배송원 웹앱의 당일 배송 백업 자료를 회계·보고 메뉴로 자동 연결했습니다.
- 거래명세가 아직 발행되지 않았더라도 다음 내용이 먼저 표시됩니다.
  · 배송 날짜
  · 호차와 배송원
  · 거래처명
  · 배송 품목
  · 품목 금액
  · 필렛 작업비
  · 전미수
  · 수금
  · 남은 미수
- 회계 확정 전 자료는 ‘배송자료·미발행’으로 표시됩니다.

3) 회계·보고 직접 수정 기능

- 최고 관리자와 회계 수정 권한이 있는 관리자는 다음 내용을 직접 수정할 수 있습니다.
  · 거래명세 번호
  · 등록 거래처
  · 거래처명
  · 호차
  · 품목 금액
  · 필렛 작업비
  · 전미수
  · 당일 수금
  · 등록·수정 사유
  · 회계 메모
- 수정할 때 기존 기록을 지우지 않고 새로운 수정 차수로 보존합니다.


==================================================
2. GitHub에 적용하는 가장 쉬운 방법
==================================================

중요:
- 이번 업데이트는 새로운 Supabase SQL을 실행할 필요가 없습니다.
- Vercel Hobby 제한을 넘지 않도록 서버 기능 파일은 11개를 유지했습니다.
- ZIP 파일 자체를 GitHub에 올리면 실행되지 않습니다.
  ZIP을 먼저 풀고, 안에 있는 파일을 GitHub의 같은 위치에 올려야 합니다.

1단계
- 제공된 ZIP 파일을 휴대폰 또는 PC에서 압축 해제합니다.

2단계
- GitHub의 Delivery_Management_System1 저장소로 들어갑니다.

3단계
- 압축을 푼 폴더 안의 파일을 GitHub의 같은 위치에 덮어씁니다.
- 특히 아래 파일이 반드시 반영되어야 합니다.
  · admin.html
  · lib/admin-workday-actions.js
  · package.json
  · package-lock.json

4단계
- GitHub에서 Commit changes를 누릅니다.

5단계
- Vercel이 자동 배포될 때까지 기다립니다.
- Vercel 배포 화면에 Ready가 나오면 완료입니다.

6단계
- 관리자 페이지에 들어갑니다.
- 오른쪽 위 ‘업데이트 새로고침’을 한 번 누릅니다.
- 화면 위 버전이 V2.5.2인지 확인합니다.


==================================================
3. 날짜 분리 확인 방법
==================================================

1) 관리자 페이지에서 ‘오늘 관제’를 누릅니다.
2) 오른쪽 날짜에서 2026-07-23을 선택합니다.
3) 7월 23일 배송 내용이 표시되는지 확인합니다.
4) 날짜를 배송이 없었던 다른 날짜로 바꿉니다.
5) 7월 23일 배송 내용이 사라지면 정상입니다.

참고:
- 선택한 날짜에 출차 승인, 긴급 발주, 전체 배송 완료 기록이 실제로 있으면
  배송지가 0곳이어도 해당 배송원은 당일 업무 대상으로 표시될 수 있습니다.


==================================================
4. 회계·보고에서 배송 이력 확인 방법
==================================================

1) 상단 메뉴에서 ‘회계·보고’를 누릅니다.
2) 오른쪽 날짜에서 배송 이력이 있는 날짜를 선택합니다.
   예: 2026-07-23
3) ‘배송 이력·거래명세 관리’ 표를 확인합니다.
4) 거래명세를 아직 확정하지 않은 배송은
   ‘배송자료·미발행’ 상태로 표시됩니다.
5) 거래처, 품목, 호차, 금액, 전미수 내용이 보이면 정상입니다.

금액이 0원으로 표시되는 경우:
- 배송원 웹앱에서 해당 품목의 단가가 입력되지 않은 경우입니다.
- ‘회계 등록·수정’을 눌러 실제 금액을 직접 입력하면 됩니다.


==================================================
5. 배송자료를 회계 확정하는 방법
==================================================

1) ‘회계·보고’ 메뉴에서 날짜를 선택합니다.
2) ‘배송자료·미발행’ 행 오른쪽의 ‘회계 등록·수정’을 누릅니다.
3) 자동으로 불러온 내용을 확인합니다.
4) 필요한 내용을 직접 수정합니다.
5) 전미수와 당일 수금을 입력합니다.
6) 등록·확정 사유와 필요한 회계 메모를 입력합니다.
7) ‘회계 확정 저장’을 누릅니다.
8) 상태가 ‘발행’으로 바뀌면 정상 저장된 것입니다.

남은 미수 계산:
전미수 + 품목 금액 + 필렛 작업비 - 당일 수금


==================================================
6. 이미 확정한 회계 내용을 다시 수정하는 방법
==================================================

1) 수정할 거래명세 행에서 ‘직접 수정’을 누릅니다.
2) 거래처명, 호차, 금액, 전미수, 수금, 메모를 수정합니다.
3) 수정 사유를 반드시 입력합니다.
4) ‘수정본 저장’을 누릅니다.
5) 아래 ‘수정·취소 이력’에서 새 수정 차수를 확인합니다.

중요:
- 기존 거래명세를 덮어쓰지 않습니다.
- 최초 기록과 수정 기록이 모두 남아 나중에 확인할 수 있습니다.


==================================================
7. Excel 저장
==================================================

1) 회계·보고 메뉴에서 원하는 날짜를 선택합니다.
2) ‘Excel 저장’을 누릅니다.
3) 확정 거래명세뿐 아니라 아직 미발행인 배송자료도 거래명세 시트에 표시됩니다.
4) 배송 품목, 회계 메모, 전미수, 수금, 남은 미수를 함께 확인할 수 있습니다.


==================================================
8. 정상 작동 확인표
==================================================

[ ] 관리자 화면 위 버전이 V2.5.2로 표시된다.
[ ] 7월 23일 배송이 다른 날짜 화면에 나오지 않는다.
[ ] 7월 23일을 선택하면 7월 23일 배송 이력이 나온다.
[ ] 회계·보고에 배송자료·미발행 행이 표시된다.
[ ] 회계 등록·수정 버튼을 눌러 금액과 내용을 수정할 수 있다.
[ ] 저장 후 발행 상태로 바뀐다.
[ ] 직접 수정 후 수정 이력이 남는다.
[ ] Excel에 배송자료와 회계 내용이 표시된다.


==================================================
9. 문제가 있을 때 확인할 내용
==================================================

1) Vercel 화면에 Ready가 표시되는지 확인합니다.
2) 관리자 페이지 오른쪽 위 ‘업데이트 새로고침’을 누릅니다.
3) 버전이 V2.5.2인지 확인합니다.
4) 회계·보고의 날짜가 배송 이력 날짜와 같은지 확인합니다.
5) 배송원 웹앱이 해당 날짜에 중앙 백업을 한 적이 있는지 확인합니다.

해당 날짜의 중앙 백업 자체가 한 번도 생성되지 않았다면,
과거 휴대폰 안에만 있던 자료를 서버에서 자동으로 복원할 수는 없습니다.
