장백수산 배송 통합 관리자 V2.5.0
전체 배송·상차 상세 및 최고 관리자 수정 기능 사용 안내

==================================================
1. 이번에 변경된 내용
==================================================

1) 인원수를 20명 또는 20개 호차로 한정하는 문구를 삭제했습니다.
   - 이제 “전체 배송원”으로 표시됩니다.
   - 배송원이 추가되면 관리자 화면의 숫자와 목록도 자동으로 늘어납니다.

2) 관리자 화면에서 불필요한 정책 설명 문구를 삭제했습니다.
   - 점포별 완료 버튼이나 개별 하차 시각에 관한 설명 문구가 나오지 않습니다.

3) 출차 승인 보드의 “상세” 화면이 확장되었습니다.
   - 거래처명과 배송 순서
   - 발주 수량과 단위
   - 품목, 규격, 원산지
   - 개별 계근값과 계근 합계
   - 수조 번호 또는 박스 운반
   - 실제 상차 완료 여부
   - 고무밴드 색상과 수량
   - 박스 계획 수량과 실제 상차 수량
   - 박스 내용과 실물 확인 여부
   - 동봉 메모, 전체 메모, 전미수
   - 당일 전체 배송 완료 시간과 완료 메모
   를 한 화면에서 확인할 수 있습니다.

4) 최고 관리자는 당일 배송·상차 내용을 직접 수정할 수 있습니다.
   - 배송 순서와 거래처명
   - 전미수와 실제 상차 박스 수
   - 긴급 추가 발주 여부
   - 품목 추가·수정·삭제
   - 개별 계근값 수정
   - 수조 번호 또는 박스 운반 변경
   - 실제 상차 완료, 계근 확정, 밴드, 품목 확인
   - 박스 추가·수정·삭제
   - 당일 전체 배송 완료 기록
   을 수정할 수 있습니다.

5) 최고 관리자는 잘못 승인한 출차 승인을 취소할 수 있습니다.
   - 승인 취소 사유를 반드시 입력해야 합니다.

6) 관리자 수정은 배송원 웹앱에 자동 반영됩니다.
   - 배송원 웹앱은 중앙 자료를 약 20초 간격으로 확인합니다.
   - 배송원에게 화면을 새로고침하도록 안내하면 더 빨리 확인할 수 있습니다.

7) 개인정보를 침해하는 기능은 추가하지 않았습니다.
   - GPS 위치 추적 기능이 없습니다.
   - 이동 경로 추적 기능이 없습니다.
   - 개인 행동 시간을 기록하는 기능을 새로 추가하지 않았습니다.
   - 업무상 필요한 전체 완료 시간과 상차·발주 자료만 사용합니다.

==================================================
2. 출차 승인 보드 상세 확인 방법
==================================================

1) 관리자 페이지에 로그인합니다.

2) 상단에서 “오늘 관제”를 누릅니다.

3) “출차 승인 보드”에서 확인할 배송원의 “상세”를 누릅니다.

4) 화면 위쪽에서 다음 내용을 확인합니다.
   - 출차 승인 상태
   - 전체 배송 상태
   - 배정 점포 수
   - 수조 상차 품목 수
   - 박스 확인 수
   - 마지막 중앙 저장 시간

5) 아래쪽 거래처 카드를 내려가며 확인합니다.
   - 각 품목의 계근값, 운반 위치, 실제 상차, 밴드
   - 박스와 동봉 내용
   - 거래처별 메모와 전미수

==================================================
3. 최고 관리자 수정 방법
==================================================

1) 관리자 페이지에서 “오늘 관제”를 누릅니다.

2) 수정할 배송원의 “상세”를 누릅니다.

3) 위쪽의 “당일 내용 전체 수정”을 누릅니다.
   - 이 버튼은 최고 관리자에게만 보입니다.

4) 필요한 내용을 수정합니다.

5) 품목을 새로 넣으려면 “＋ 품목 추가”를 누릅니다.

6) 품목을 없애려면 해당 품목의 “품목 삭제”를 체크합니다.

7) 개별 계근값은 쉼표로 구분하여 입력합니다.
   예: 1.250, 1.330, 1.180

8) 수조를 사용하지 않고 박스로 운반하면
   “운반 위치”에서 “박스 운반”을 선택합니다.

9) 박스를 새로 넣으려면 “＋ 박스 추가”를 누릅니다.

10) 당일 전체 배송 완료 기록을 만들거나 수정하려면
    “당일 전체 배송 완료 기록”을 체크하고
    완료 결과, 시간, 메모를 입력합니다.

11) 화면 아래 “수정 사유”를 반드시 입력합니다.
    예: 현장 확인 후 광어 2마리 계근값과 3번 수조를 정정

12) “전체 내용 저장”을 누릅니다.

13) 저장이 끝나면 상세 화면으로 돌아오며,
    아래 “관리자 보정 이력”에서 수정 사유를 확인할 수 있습니다.

==================================================
4. 출차 승인 취소 방법
==================================================

1) 출차 승인된 배송원의 “상세”를 누릅니다.

2) “출차 승인 취소”를 누릅니다.

3) 승인 취소 사유를 입력합니다.

4) 확인 후 상차 내용을 다시 점검합니다.

5) 문제가 해결되면 출차 승인 보드에서 다시 승인합니다.

==================================================
5. GitHub와 Vercel에 적용하는 방법
==================================================

중요: 이번 버전은 admin.html 한 파일만 바꾸면 안 됩니다.
아래 파일과 폴더를 모두 올려야 최고 관리자 수정 기능이 작동합니다.

필수 업로드 대상

- admin.html
- driver.html
- package.json
- package-lock.json
- api/admin/update-driver-workday.js
- api/admin/reset-departure.js

가장 쉬운 방법

1) 제공된 전체 프로젝트 ZIP 파일의 압축을 풉니다.

2) GitHub의 Delivery_Management_System1 저장소에 들어갑니다.

3) “Add file”을 누르고 “Upload files”를 누릅니다.

4) 압축을 푼 프로젝트의 파일과 api 폴더를 올립니다.
   기존 파일이 있으면 같은 이름으로 덮어씁니다.

5) 아래쪽 “Commit changes”를 누릅니다.

6) Vercel이 자동 배포를 시작합니다.

7) Vercel에서 초록색 “Ready”가 나오면 배포가 끝난 것입니다.

8) 관리자 페이지에 접속한 뒤 상단의
   “업데이트 새로고침” 버튼을 누릅니다.

9) 관리자 화면 상단 버전이 V2.5.0인지 확인합니다.

==================================================
6. 정상 작동 확인
==================================================

1) 관리자 페이지 상단에 V2.5.0이 표시됩니다.

2) “20개 호차”라는 고정 문구가 보이지 않습니다.

3) 출차 승인 보드의 “상세”를 누르면
   품목·계근·수조·밴드·박스 내용이 보입니다.

4) 최고 관리자에게만 “당일 내용 전체 수정”이 보입니다.

5) 내용을 수정하고 저장하면 성공 안내가 나옵니다.

6) 배송원 웹앱을 새로고침하면 수정한 내용이 보입니다.

오류가 나오면 오류 화면을 캡처하고,
어떤 버튼을 누른 뒤 발생했는지 함께 확인해야 합니다.
