장백수산 배송 통합 관리자 V2.5.3
배송 없는 날짜에 과거 자료가 표시되는 오류 수정 안내

==================================================
1. 확인된 원인
==================================================

2026년 7월 25일에 실제 배송은 없지만 아래와 같은 빈 기록이 미리 생성되어 있었습니다.

- 배송원 중앙 백업 기록
- 출차 점검 기록
- 이전 상태가 남아 있는 배송원 상태 기록
- 과거 시각으로 표시되는 마지막 동기화 기록

기존 프로그램은 실제 배송지가 없어도 위 기록 중 하나가 있으면
‘당일 업무 배송원’으로 잘못 판단했습니다.

화면에서 다음 내용이 동시에 나온 것이 이 오류의 근거입니다.

- 당일 배정 0곳
- 전체 완료 시간 없음
- 마지막 동기화는 과거 날짜
- 그런데도 당일 업무 배송원 1명으로 표시


==================================================
2. V2.5.3에서 변경된 기준
==================================================

이제 다음 조건 중 하나가 있을 때만 당일 업무 배송원으로 표시됩니다.

1) 선택한 날짜에 실제 배송지가 1곳 이상 등록되어 있는 경우
2) 선택한 날짜에 해당 배송원의 긴급 발주가 등록되어 있는 경우
3) 실제 배송지가 있었고 전체 배송 완료 기록이 있는 경우

다음 기록만 있는 경우에는 당일 업무로 표시하지 않습니다.

- 빈 중앙 백업
- 배송지 0곳인 상태 기록
- 출차 점검 기록만 존재
- 과거 마지막 동기화 시각
- 배송지가 없는 전체 완료 빈 기록


==================================================
3. 정상 화면
==================================================

배송이 없는 2026년 7월 25일을 선택하면 다음과 같이 표시됩니다.

- 등록 배송원: 기존 등록 인원
- 당일 업무 배송원: 0
- 출차 승인: 0/0
- 전체 배송 완료: 0
- 출차 승인 보드: 2026-07-25에 등록된 배송 업무가 없습니다.
- 전체 배송 완료 현황: 2026-07-25에 등록된 배송 업무가 없습니다.
- 당일 중앙 백업: 0/0호차
- 상단 안내: 2026-07-25에 등록된 배송 업무가 없습니다.

해당 날짜의 실제 발주 또는 긴급 발주가 등록되면 자동으로 다시 표시됩니다.


==================================================
4. 적용 방법
==================================================

1) V2.5.3 ZIP 파일을 다운로드합니다.
2) ZIP 파일을 먼저 압축 해제합니다.
3) 압축을 푼 파일을 GitHub 저장소의 같은 위치에 덮어씁니다.
4) Commit changes를 누릅니다.
5) Vercel 배포 상태가 Ready가 될 때까지 기다립니다.
6) 관리자 페이지에서 오른쪽 위 ‘업데이트 새로고침’을 누릅니다.
7) 화면 위 버전이 V2.5.3인지 확인합니다.

중요:
- ZIP 파일 자체만 GitHub에 올리면 안 됩니다.
- ZIP 안의 파일을 압축 해제한 뒤 올려야 합니다.
- 이번 수정은 새로운 Supabase SQL을 실행하지 않습니다.


==================================================
5. 확인 방법
==================================================

1) 관리자 페이지에서 ‘오늘 관제’를 누릅니다.
2) 실제 배송이 없는 날짜를 선택합니다.
3) ‘당일 업무 배송원’이 0인지 확인합니다.
4) Daniel 배송원 카드가 나오지 않는지 확인합니다.
5) 배송 이력이 있는 날짜로 변경합니다.
6) 해당 날짜의 배송원 카드가 정상적으로 나오는지 확인합니다.
