장백수산 배송 중앙관리 시스템 설치 안내

구성 파일
------------------------------------------------------------
driver.html
  배송원 로그인, 관리자 호차 자동 배정, 중앙 상태·백업 동기화

admin.html
  배송원 계정 생성, 호차 변경, 계정 정지, 진행 현황 확인

api/config.js
  배송원·관리자 화면에 안전하게 공개할 연결 정보 전달

api/admin/create-driver.js
  관리자 권한으로 배송원 로그인 계정 생성

api/admin/reset-password.js
  배송원 임시 비밀번호 초기화

supabase-schema.sql
  공용 데이터베이스 표, 권한, 보안 규칙 생성

package.json / vercel.json
  Vercel 서버 기능과 캐시 방지 설정


1단계 · Supabase 프로젝트 생성
------------------------------------------------------------
1. https://supabase.com/dashboard 에 접속합니다.
2. New project를 눌러 프로젝트를 만듭니다.
3. 데이터베이스 비밀번호를 안전한 곳에 별도로 보관합니다.
4. 프로젝트 생성이 끝나면 SQL Editor를 엽니다.
5. supabase-schema.sql 전체 내용을 복사해 실행합니다.


2단계 · 최초 관리자 계정 생성
------------------------------------------------------------
1. Supabase → Authentication → Users를 엽니다.
2. Add user를 누릅니다.
3. 실제로 사용할 관리자 이메일과 비밀번호를 입력합니다.
4. 이메일 확인 완료 상태로 계정을 생성합니다.
5. supabase-schema.sql 마지막 부분의
   admin@example.com을 방금 만든 관리자 이메일로 바꿉니다.
6. "최초 관리자 지정" INSERT 문장만 SQL Editor에서 다시 실행합니다.

주의: 관리자 비밀번호는 코드, GitHub, 카카오톡 단체방에 적지 않습니다.


3단계 · Supabase 키 확인
------------------------------------------------------------
Supabase 프로젝트 설정의 API Keys 화면에서 다음 값을 확인합니다.

1. Project URL
2. Publishable key 또는 anon public key
3. service_role key

service_role key는 절대로 HTML이나 GitHub 코드에 입력하면 안 됩니다.
Vercel 환경변수에만 저장합니다.


4단계 · GitHub에 파일 업로드
------------------------------------------------------------
현재 Delivery_Management_System1 저장소의 최상위 위치에 다음 파일과
폴더를 올립니다.

driver.html
admin.html
package.json
vercel.json
api 폴더 전체

기존 driver.html은 수정본으로 교체합니다.
supabase-schema.sql과 설치안내.txt는 보관용으로 함께 올려도 됩니다.

GitHub의 main 브랜치에서 Commit changes를 누릅니다.


5단계 · Vercel 환경변수 등록
------------------------------------------------------------
1. https://vercel.com/dashboard 에 접속합니다.
2. delivery-management-system1 프로젝트를 엽니다.
3. Settings → Environment Variables를 엽니다.
4. 아래 네 개를 추가합니다.

SUPABASE_URL
  Supabase Project URL

SUPABASE_ANON_KEY
  Supabase Publishable key 또는 anon public key

SUPABASE_SERVICE_ROLE_KEY
  Supabase service_role key

DRIVER_EMAIL_DOMAIN
  drivers.invalid

각 환경변수는 Production, Preview, Development에 적용합니다.
저장한 뒤 반드시 새로 Redeploy 해야 합니다.


6단계 · Vercel 재배포
------------------------------------------------------------
1. Vercel → Deployments를 엽니다.
2. 최신 main 커밋이 자동 배포됐는지 확인합니다.
3. 자동 배포가 없다면 Redeploy를 실행합니다.
4. 상태가 Ready가 될 때까지 기다립니다.


7단계 · 관리자 페이지 접속
------------------------------------------------------------
https://delivery-management-system1.vercel.app/admin.html

Supabase에서 만든 관리자 이메일과 비밀번호로 로그인합니다.

관리자 페이지에서 배송원 등록을 누르고 다음 내용을 입력합니다.

배송원 아이디 예시: driver01
배송원 이름: 실제 이름
배정 호차: 1
임시 비밀번호: 8자리 이상

배송원 아이디는 영문 소문자, 숫자, 밑줄, 하이픈만 사용합니다.


8단계 · 배송원 로그인
------------------------------------------------------------
https://delivery-management-system1.vercel.app/driver.html

배송원은 관리자가 만든 아이디와 임시 비밀번호를 입력합니다.
로그인하면 관리자가 배정한 이름과 호차가 자동 적용됩니다.

배송원은 설정에서 이름이나 호차를 직접 변경할 수 없습니다.
관리자가 호차를 변경하면 배송원 화면에 자동 반영됩니다.


9단계 · 1호차부터 20호차 배포 순서
------------------------------------------------------------
1. 관리자 본인과 배송원 1명으로 먼저 시험합니다.
2. 로그인, 호차 표시, 배송 자료 저장, 관리자 현황을 확인합니다.
3. 문제가 없으면 2~5호차에 배포합니다.
4. 하루 이상 확인 후 6~20호차에 배포합니다.

배송원에게 전달할 내용

[장백수산 배송원 프로그램]
주소: https://delivery-management-system1.vercel.app/driver.html
아이디: 관리자가 개별 전달
임시 비밀번호: 관리자가 개별 전달

아이디와 비밀번호는 단체방에 올리지 않고 배송원에게 각각 전달합니다.

보안 주의
------------------------------------------------------------
SUPABASE_SERVICE_ROLE_KEY는 GitHub에 올리지 않습니다.
관리자 비밀번호를 배송원과 공유하지 않습니다.
배송원이 퇴사하면 관리자 페이지에서 즉시 사용 정지합니다.
GitHub가 공개 저장소여도 환경변수 값은 코드에 쓰지 않습니다.
실제 운영 전 반드시 1~2명으로 시험합니다.
