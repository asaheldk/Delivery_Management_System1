장백수산 배송 중앙관리 V2 · 6대 기능 업데이트 설치 안내
=========================================================

중요: 아래 순서를 지키면 기존 배송원 계정과 당일 자료가 사라지지 않습니다.

[업데이트 전에]
1. GitHub 저장소의 현재 파일을 별도 폴더에 내려받아 보관합니다.
2. 배송 중인 시간이 아니라 모든 배송원이 업무를 마친 뒤 업데이트합니다.
3. Supabase와 Vercel의 기존 환경변수 3개는 삭제하거나 바꾸지 않습니다.

[1단계 · Supabase 표와 권한 추가]
1. Supabase 사이트에 로그인합니다.
2. 왼쪽 메뉴 SQL Editor를 누릅니다.
3. New query를 누릅니다.
4. 이 폴더의 supabase-schema.sql 전체를 복사해 붙여넣습니다.
5. 오른쪽 아래 Run을 한 번 누릅니다.
6. Success가 나오면 완료입니다.

이 SQL은 기존 V1에 표와 권한을 추가하는 방식입니다.
기존 admin 계정은 계속 전체 관리자 권한을 가집니다.

[2단계 · GitHub 파일 교체]
GitHub 저장소 Delivery_Management_System1의 main 브랜치에서 아래 파일을 같은 위치에 올립니다.

교체할 파일
- admin.html
- driver.html
- supabase-schema.sql
- package.json
- vercel.json
- api/admin/auth.js
- api/admin/create-driver.js
- api/admin/reset-password.js

새로 추가할 파일
- api/admin/update-driver.js
- api/admin/create-staff.js
- api/admin/update-role.js
- api/admin/store-secret.js

파일 위치가 중요합니다. 예를 들어 store-secret.js는 반드시
api 폴더 → admin 폴더 안에 있어야 합니다.

[3단계 · Vercel 재배포 확인]
1. GitHub에서 마지막 파일 저장을 완료합니다.
2. Vercel 프로젝트 → Deployments로 이동합니다.
3. 가장 위 배포가 Ready가 될 때까지 기다립니다.
4. 실패하면 배포 오른쪽 점 3개 → Redeploy를 누릅니다.

[4단계 · 관리자 확인]
주소: https://delivery-management-system1.vercel.app/admin.html

1. 기존 관리자 이메일과 비밀번호로 로그인합니다.
2. 상단에 다음 7개 메뉴가 표시되는지 확인합니다.
   오늘 관제 / 배송원·출차 / 거래처·인수인계 / 긴급·오류 /
   품목·가격 / 회계·보고 / 권한·감사
3. 거래처 한 곳과 품목 한 개를 시험 등록합니다.
4. 긴급 발주를 시험 등록한 뒤 배송원 화면을 확인합니다.

[5단계 · 배송원 확인]
주소: https://delivery-management-system1.vercel.app/driver.html

1. 시험 배송원 계정으로 로그인합니다.
2. 배송 화면 맨 위에 ‘중앙 관리자 연결’ 상자가 보이는지 확인합니다.
3. 시험 긴급 발주에서 ‘긴급 발주 확인’을 누릅니다.
4. 관리자 ‘긴급·오류’ 메뉴에서 상태가 바뀌었는지 확인합니다.
5. 시험 자료는 관리자 화면에서 취소 상태로 바꿉니다.

[각 메뉴 사용 위치]

1순위 반드시 추가 기능
- 오늘 관제: 20개 호차 출차 승인, 당일 마감 잠금, 중앙 백업 상태
- 배송원·출차: 호차 중복 차단, 호차 배정 이력, 퇴사·이동 사유

2순위 거래처·인수인계
- 거래처·인수인계: 주소, 연락처, 진입 방법, 놓는 자리, 금지사항,
  입고 시간, 주차, 기타 인계, 담당 호차와 순서
- 인수인계표 버튼: 새 배송원에게 전달할 인쇄용 화면
- 비밀번호 버튼: 최고 관리자·배차 관리자만 열람하고 감사 기록을 남김

3순위 긴급 발주·오류
- 긴급·오류: 긴급 발주 등록과 확인/상차/납품 상태
- 배송원 화면 맨 위: 긴급 발주 처리 버튼과 오류 보고 버튼

4순위 품목·가격
- 품목·가격: 활어/패류/냉동/필렛, kg/마리/팩/박스 단위,
  원산지, 기본 단가, 필렛 작업비, 밴드 색, 적용일
- 거래처 단가 버튼: 특정 거래처에만 적용할 가격 기록

5순위 회계·보고
- 회계·보고: 품목금액, 필렛 작업비, 전미수, 당일 수금, 남은 미수
- 수정본: 기존 명세를 지우지 않고 2차, 3차 수정본으로 보존
- Excel 저장: 거래명세, 호차별 배송, 품목, 미수, 취소·수정,
  긴급 발주, 오류 보고를 한 파일로 저장

6순위 관리자 권한
- 권한·감사: 최고/배차/회계/현장/조회 전용 관리자 계정
- 감사 기록: 등록, 수정, 취소, 마감, 비밀번호 열람 이력

[권장 첫 시험]
실제 20명 전체에게 바로 적용하지 말고 관리자 1명 + 배송원 1명으로
하루 시험한 뒤, 2~3명, 전체 순서로 확대하세요.

[문제가 생겼을 때]
1. 관리자 화면이 비어 있으면 supabase-schema.sql을 Run했는지 확인합니다.
2. 관리자 권한 오류는 profiles 표에서 본인 role이 admin 또는 super_admin인지 확인합니다.
3. 긴급 발주가 배송원에게 안 보이면 거래처의 배정 배송원·호차가 맞는지 확인합니다.
4. 배송원 수정이 안 되면 오늘 관제에서 당일 업무가 마감되었는지 확인합니다.
5. Vercel이 실패하면 Deploy Logs의 빨간 줄 화면을 캡처해 확인합니다.
